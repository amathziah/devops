# Contributing to ShopSmart

Thank you for your interest in contributing to ShopSmart! We welcome contributions from everyone.

## 🚀 How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally: `git clone https://github.com/[your-username]/devops.git`.
3.  **Create a branch** for your feature or bugfix: `git checkout -b my-new-feature`.
4.  **Install dependencies**:
    -   Frontend: `cd client && npm install`
    -   Backend: `cd server && npm install`
5.  **Make your changes**.
6.  **Run tests** to ensure no regressions: `npm run test:all`.
7.  **Commit your changes** with descriptive commit messages.
8.  **Push to your fork**: `git push origin my-new-feature`.
9.  **Submit a Pull Request** to the `main` branch.

## 🧪 Testing Requirements

All contributions must include relevant tests:
-   **Unit Tests**: For isolated component or utility logic.
-   **Integration Tests**: For API endpoints and database interactions.
-   **E2E Tests**: For significant UI flow changes.

## 📜 Code Style

-   Follow Prettier and ESLint rules.
-   Maintain high test coverage (80%+).
-   Use descriptive variable and function names.
